Case Study: WWOZ Livewire

New Orleans Music Calendar

Creating a mobile guide to live local music in and around New Orleans, featuring venue and artist information from WWOZ's music calendar.

Client:
WWOZ New Orleans
Platform:
iOS
Timeline:
3 months
Role:
Lead Developer

Project Results

The Challenge

WWOZ, a renowned community radio station in New Orleans, needed a mobile solution to share their extensive "Livewire Music Calendar" with local and visiting music enthusiasts. The project presented several key challenges:

  • Dynamic data presentation: Finding intuitive ways to organize and display hundreds of daily music events across the city
  • Complex information architecture: Creating relationships between venues, artists, and performances
  • Local navigation needs: Helping users discover music in New Orleans' unique geography
  • Offline functionality: Ensuring the app remained useful while tourists were exploring without reliable data connections
  • Preserving the WWOZ brand: Maintaining the station's distinctive community-focused identity in a digital format

My Approach

After collaborative discovery sessions with the WWOZ team, I developed a user-centered approach focused on both tourists and locals:

Experience Design:

  • Time-based navigation: Prioritizing "today's music" as the primary entry point for users
  • Dual discovery paths: Allowing users to browse both by venue and by performer
  • User preference tracking: Enabling users to follow favorite acts and learn about upcoming performances
  • Rich content integration: Incorporating videos, social media links, and artist information

Development Approach:

I leveraged a rapid development methodology with frequent stakeholder feedback loops to ensure the app met both user and client expectations:

  • API-first implementation: Building a robust communication layer with WWOZ's calendar system
  • Offline caching strategy: Designing a system that would store recent queries for offline use
  • Performance optimization: Ensuring smooth scrolling and transitions despite large dataset
  • Collaborative UI design: Working with WWOZ's designer to create an interface that felt true to the brand

Technical Implementation

Developing the WWOZ Livewire app required several technical solutions to address the project's unique requirements:

iOS Native Development:

I built a native iOS application to ensure the best possible performance and user experience:

  • Custom calendar interface optimized for music events
  • Location-aware venue discovery using CoreLocation
  • Efficient caching system for offline browsing
  • Dynamic list management for handling large datasets smoothly
  • Optimized image handling for artist and venue photos

Data Architecture:

The app's data structure was carefully designed to handle the complex relationships between events, venues, and artists:

  • Real-time API integration with WWOZ's events database
  • Intelligent caching layer for both performance and offline access
  • Custom search indexing for quick lookup by various criteria
  • Favorite synchronization across app sessions

Media Integration:

A key feature was connecting users with artists' content beyond just event listings:

  • YouTube video integration for artist performances
  • Social media profile linking to connect with artists
  • Website deep linking for venues and performers

Technologies Used:

  • Objective-C and UIKit for iOS development
  • REST API integration with caching layer
  • CoreLocation for mapping and proximity features
  • YouTube Player integration
  • Custom persistence layer for favorites

Key Learnings

This project provided valuable insights into developing location-based event apps:

User Experience Insights:

  • Contextual relevance is crucial - Users primarily wanted to know "what's happening tonight near me"
  • Multiple discovery paths - Different user groups (tourists vs. locals, venue-focused vs. artist-focused) needed different navigation options
  • Offline functionality matters - Visitors with limited data plans or in areas with poor reception still needed access to information
  • Simplicity wins - Despite the complex data relationships, the interface needed to remain straightforward

Technical Learnings:

  • Data optimization is essential - Event apps require careful consideration of how to structure and cache data
  • Location services add complexity - Map integration and geolocation features required additional performance considerations
  • Content freshness balance - Finding the right balance between real-time updates and cached content
  • Media integration tradeoffs - Incorporating rich media while maintaining performance required careful implementation

These learnings have informed my approach to subsequent event and location-based applications, particularly in how I structure data and create contextually relevant user experiences.

If I Were Building This Today

If I were developing the WWOZ Livewire app today, I would leverage several AI capabilities to significantly enhance the user experience:

Personalized Recommendations:

  • Music Taste Prediction - Using machine learning to analyze user behavior and suggest events based on their musical preferences
  • Smart Itinerary Creation - AI-powered tools to build optimized music crawls based on location, timing, and preferences
  • Sentiment Analysis - Processing reviews and social media to highlight events generating the most positive buzz
  • Personalized Notifications - ML models that learn when and how often to notify users about relevant events

Enhanced Discovery:

  • Music Style Classification - Automatically categorizing artists by genre and style based on audio samples and descriptions
  • Similar Artist Suggestions - "If you like this artist, you might also enjoy..." recommendations
  • Natural Language Search - Conversational interfaces to find events (e.g., "Find me jazz near Frenchmen Street tomorrow night")
  • Image Recognition - Using venue photos to understand the atmosphere and match to user preferences

Content Enrichment:

  • Automated Artist Profiles - Using AI to gather and synthesize information about performers from across the web
  • Music Preview Generation - Creating brief audio previews of an artist's style using generative AI
  • Automated Event Summaries - Generating concise descriptions of events based on venue, performer, and historical data
  • Cross-Platform Content Aggregation - Automatically collecting and featuring artist videos, reviews, and social media content

These AI enhancements would transform what was initially a digital event calendar into an intelligent music concierge that understands each user's unique preferences and helps them discover exactly the right musical experiences in New Orleans. The technology would amplify the human element rather than replace it—making it easier for people to connect with the authentic cultural experiences the city has to offer.
